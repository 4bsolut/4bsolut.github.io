Hexagonal architecture in real projects

Published February 5, 2024

Hexagonal architecture keeps your domain independent from frameworks and external services. In Spring Boot, this means the core stays clean while adapters handle infrastructure concerns.

Start with the domain

Define entities, value objects, and use cases without depending on Spring annotations. Keep business logic in plain classes.

Ports before adapters

Create interfaces for persistence, messaging, and external APIs. Implement adapters in the infrastructure layer and wire them through configuration.

Testing becomes simpler

With ports, you can test use cases without a database. Use in-memory adapters or mocks to validate logic quickly.

Common pitfalls

  • Leaking frameworks into domain classes.
  • Mixing HTTP models with domain models.
  • Skipping contract tests between ports and adapters.

When done well, hexagonal architecture reduces coupling and makes delivery teams more autonomous.
